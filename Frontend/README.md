# VOLT Storefront (Frontend)

Customer-facing e-commerce storefront: product catalog, cart, checkout (Razorpay), auth, profile, and wishlist. Built as a Vite SPA with TanStack Router.

**Live URL:** https://volt-frontend-x4ed.onrender.com

## Tech stack

- React 19 + TypeScript
- Vite 7
- TanStack Router (file-based routes)
- TanStack Query
- Tailwind CSS 4
- Radix UI + shadcn-style components
- Framer Motion
- Razorpay (checkout)

## Configuration

This app does **not** use `.env` files. Public settings live in:

```ts
// Frontend/env.public.ts
export const publicEnv = {
  VITE_FRONTEND_URL: "https://volt-frontend-x4ed.onrender.com",
  VITE_API_URL: "https://volt-backend-20cc.onrender.com",
  VITE_BACKEND_ORIGIN: "https://volt-backend-20cc.onrender.com",
  VITE_RAZORPAY_KEY_ID: "rzp_test_Snh6GjGsBzuwL6",
} as const;
```

| Variable | Purpose |
|----------|---------|
| `VITE_FRONTEND_URL` | Public storefront URL |
| `VITE_API_URL` | Backend API base (browser calls this directly in production) |
| `VITE_BACKEND_ORIGIN` | Backend origin for Vite dev proxy and SSR fallbacks |
| `VITE_RAZORPAY_KEY_ID` | Razorpay public key for checkout (test key shown) |

Update `env.public.ts`, then restart dev or rebuild.

## Local development

```bash
cd Frontend
npm install
npm run dev
```

Vite serves the app (default port from the toolchain, often 5173 or 8080). API calls use `VITE_API_URL` when set, or the dev proxy to `VITE_BACKEND_ORIGIN` for `/api/*`.

## Deploy on Render

Use a **Static Site** with:

| Setting | Value |
|---------|--------|
| Build Command | `cd Frontend && npm install && npm run build` |
| Publish Directory | `Frontend/dist` |
| Rewrite | `/*` → `/index.html` |

Or use the root `render.yaml` service `volt-frontend`.

**Important:** The build must produce `Frontend/dist/index.html`. Do not publish the repo root or a server-only output folder.

Redeploy the backend after changing `FRONTEND_URL` so CORS allows https://volt-frontend-x4ed.onrender.com.

## Backend integration

- Products: `GET /api/products` (query: `category`, `clothType`, `sort`, `page`, `limit`)
- Catalog filters: `GET /api/catalog/cloth-types`
- Auth: `/api/auth/*`
- Cart / orders / payments: `/api/cart`, `/api/orders`, `/api/payments`

API base URL is built in `src/lib/api.ts` via `resolveApiUrl()`.

## Troubleshooting

| Issue | What to check |
|-------|----------------|
| Render shows plain **Not Found** | Publish `Frontend/dist`, add SPA rewrite `/*` → `/index.html` |
| **0 products** on `/products` | Backend up, MongoDB seeded, CORS includes frontend URL |
| Razorpay not configured | Set `VITE_RAZORPAY_KEY_ID` in `env.public.ts`; secret stays on backend |
| API returns HTML instead of JSON | `VITE_BACKEND_ORIGIN` / `VITE_API_URL` points to the API, not the storefront |
| CORS blocked | Backend `FRONTEND_URL` in `backend/config/appEnv.production.ts` |
